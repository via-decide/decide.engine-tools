# AGENTS.md — repository rules

Scope: entire repository.

## Mission

Treat this repo as a preservation-first staging mesh and Orchard Engine implementation workspace.

## Non-destructive requirements

- Never delete unrelated files or folders.
- Never reduce existing tool capability to force architectural consistency.
- Never replace real tools with placeholders.
- If migration is partial, preserve behavior and add adapters/metadata.

## Workflow requirements

1. Audit before editing.
2. Identify user-facing tools and preserve existing behavior.
3. Prefer additive updates in shared utilities and tool folders.
4. Keep standalone HTML tools runnable locally.
5. Build Orchard Engine iteratively (Wave-by-Wave), not all-at-once.
6. Return complete contents of edited files when requested.

## Orchard-specific rules

- Keep progression merit-based (no pay-to-win logic).
- Preserve metaphor mapping in UI where practical (roots, trunk, branches, leaves, fruits, seeds, water, minerals, soil, sunlight).
- Keep fairness and anti-spam controls explicit in ranking and promotion tools.
- Keep AI/advisory logic decoupled from critical scoring loops.

## Output discipline

- Do not omit touched files.
- Do not use destructive rewrites for unrelated code.
- Prefer modular vanilla HTML/CSS/JS.
- Do not remove existing tool folders just to fit new architecture.
