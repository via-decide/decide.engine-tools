# Simulation Marketplace

## Overview

The simulation marketplace pipeline allows simulations generated by users or Zayvora to be packaged and published to daxini.space.

## Manifest Format

Each simulation must generate a manifest before packaging.

Required fields:

- `name`
- `creator`
- `engine`
- `entry`
- `version`
- `assets`
- `created_at`

Example:

```json
{
  "name": "Mars Colony Economy",
  "creator": "user_id",
  "engine": "decide-engine",
  "entry": "games/mars/game-entry.js",
  "version": "1.0",
  "assets": ["games/mars/config/config.json"],
  "created_at": "2026-04-20T00:00:00.000Z"
}
```

## Packaging Pipeline

`marketplace/simulation-packager.js` performs:

1. Script collection from simulation definitions
2. Asset collection from simulation definitions and config
3. Manifest generation through `SimulationManifest.buildManifest()`
4. Package generation with output path format `dist/<simulation-name>.pkg`

## Security Validation

`marketplace/security-check.js` runs the publishing gate:

- `verifyCompatibility()`
- `validateEntryPoint()`
- `scanAssets()`
- `verifyManifestIntegrity()`

Publishing stops when any check fails.

## Publish Workflow

`marketplace/publish-simulation.js` orchestrates:

1. Build simulation package
2. Generate manifest
3. Package assets
4. Create launch URL (`https://daxini.space/simulations/<slug>`)
5. Register simulation metadata in marketplace APIs

## Daxini Integration

`marketplace/marketplace-client.js` provides API operations:

- register simulation (`POST /marketplace/register`)
- send metadata (`POST /marketplace/metadata`)
- update simulation info (`POST /marketplace/update`)
- fetch marketplace apps (`GET /marketplace/apps`)

Retry logic is included for post requests.

## Launch Process

Published simulations produce launch URLs that resolve into the Decide engine runtime via shared module metadata.

The exporter API:

- `SimulationPublisher.exportSimulation(name, options)`

supports both generated simulations and the baseline Mars simulation fallback.

## Workspace Publish Integration

The publish trigger is modular and can be called from workspace flows:

User builds simulation → simulation exported and packaged → publish pipeline executes → simulation discoverable on daxini.space.
