- Use: Decision Brief, Sales Dashboard, Workflow Builder, Revenue Forecaster
- Task: "Create a plan" → Start with Decision Brief template

**Testing & Validation**
- Use: Output Evaluator, Prompt Compare, Tool Search Discovery
- Task: "Compare approaches" → Use evaluation tools

**Automation & Workflow**
- Use: Workflow Builder, Task Splitter, Template Vault
- Task: "Automate X" → Check workflow patterns first

### Token Economy Benefits:
- ✅ 81-90% fewer tokens per feature (use existing patterns)
- ✅ 10x faster implementation (patterns compound)
- ✅ Consistent quality (battle-tested templates)
- ✅ Zero setup required (browser-native, no build step)

### My Workflow:
1. **Understand the task** — What does the user want?
2. **Check decide.engine-tools** — Is there a pattern/tool for this?
3. **Reference existing patterns** — Use metadata from the repo
4. **Build additive** — Extend, don't duplicate
5. **Document decisions** — Help future patterns compound

### Tool Categories Quick Reference:
- **👨‍💻 Coders**: Code Generator, Code Reviewer, Agent Builder, App Generator
- **🎨 Creators**: Prompt Alchemy, Script Generator, Design System Tools
- **🔬 Researchers**: Multi-Source Research, Student Research, Data Analysis
- **📚 Learners**: Interview Prep, Study OS, Knowledge Graph
- **💼 Professionals**: Decision Brief, Sales Dashboard, Workflow Builder
- **🎮 Gaming**: Orchard Engine, Wings of Fire Quiz

### Key Principles:
- **Metadata Matters**: Every artifact captured = exponential future savings
- **Patterns Over Pixels**: Clone → Analyze → Adapt (don't build from scratch)
- **Preservation First**: New tools additive, never breaks existing functionality
- **Browser Native**: No build step, no npm install, no config hell

### When I'm Not Using Tools:
Only if the task is genuinely novel and no pattern exists. Then:
- Build a clean, reusable solution
- Document the pattern
- Consider contributing it back to the repo
